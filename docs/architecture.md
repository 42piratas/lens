# Architecture

LENS is built around **five modular surfaces**. Almost every contribution adds an instance of one of them. The surfaces are folder-modular: each instance lives in its own directory, registers via codegen, and never requires editing a central registry by hand.

| Surface | Folder | Adds | Cookbook |
|:--|:--|:--|:--|
| **Connectors** | [`app/connectors/<id>/`](../app/connectors/README.md) | A data source (Calendar, Trello, Sheets, …) — manifest + auth + client + per-tile adapters + API route handlers | [add-a-connector](contrib/add-a-connector.md) |
| **Tiles** | [`app/tiles/<id>/`](../app/tiles/README.md) | A way to render data — manifest + Component + optional shared `TileAdapter` | [add-a-tile](contrib/add-a-tile.md) |
| **Themes** | [`app/themes/<id>/`](../app/themes/README.md) | A palette + `tokens.css` over the semantic-token layer | [add-a-theme](contrib/add-a-theme.md) |
| **Workspaces** | [`app/lib/workspace/`](../app/lib/workspace/README.md) | (No code surface — UI configuration; saved layout + theme snapshots per workspace) | — |
| **Plugins** | [`app/lib/dnd-payloads/`](../app/lib/dnd-payloads/README.md) + per-connector `payload-adapters/` | A new cross-tile drag-payload kind, or a new absorber for an existing kind | [add-a-plugin-payload](contrib/add-a-plugin-payload.md) |

## Connectors

A connector provides **data**. Each connector is a self-contained folder under `app/connectors/<id>/` with a single manifest, an auth strategy, a server-side client, optional per-tile adapters, and route handlers.

Five connector shapes ship today:

- **External + OAuth** — Google Calendar, Google Sheets, Google Tasks (per-user OAuth via Auth.js v5; tokens encrypted in Supabase Vault).
- **External + API key** — Trello (key + token), Trakt (`TRAKT_CLIENT_ID`).
- **External + auth-free** — Goodreads (per-shelf RSS, public profiles only).
- **External + service-account / domain-wide delegation** — Google Keep (Workspace-only; the Keep API isn't in the user-OAuth scope catalog, so a Workspace admin grants the SA `keep.readonly` and LENS impersonates `session.user.email`).
- **Local** — Scratchpad (`localStorage`-facade with server-side sync).

Every connector exposes a `ConnectorManifest` whose `id` literal must match the folder name. The registry is auto-generated by `pnpm gen:registries` from disk state — never hand-edit `app/connectors/index.ts`.

## Tiles

A tile **renders** data. Tiles live under `app/tiles/<id>/` and can be either single-connector (a calendar tile knows about Calendar) or shared across compatible connectors (a `task-list` tile renders both Trello and Google Tasks rows via per-connector `TileAdapter` registrations).

The 14 tile types span calendar views, kanban boards, task lists, data displays (table / chart / KPI), media lists, note buffers, and badge rows. Each tile exposes a `TileManifest` whose `id` must match the folder name.

## Themes

A theme is a palette. Each theme lives under `app/themes/<id>/` with a `manifest.ts` and a `tokens.css` defining every required semantic token under `[data-theme="<id>"]`. `globals.css` imports an auto-generated `app/themes/index.css` that aggregates every theme's tokens.

Selection is via `<html data-theme="<id>">`, set by the Dock toggle and the `/settings` page. Bundled themes include 42labs DS Light/Dark, Catppuccin (Latte/Mocha), Tokyo Night (Day/Storm), Solarized, Rosé Pine, Gruvbox, One, Monokai, Nord, Dracula, GitHub Light, Ayu Light, and more.

## Workspaces

Workspaces are saved snapshots of the entire dashboard state — layout + active theme. One install holds N workspaces; the operator switches via the Dock. Switching is atomic.

Schema: `{ id, name, icon, theme, layout: LayoutCard[] }`. Persisted to `localStorage["lens.workspaces"]` with bidirectional server-side sync via `lib/workspace/SyncBridge.tsx`.

Workspaces are not a code surface — adding one is a UI action.

## Plugins (cross-tile drag-payloads)

Plugins are how data crosses tile boundaries. Drag a producer (e.g. a Sheets badge), drop on a foreign connector's tile, the foreign connector absorbs the payload into its native model.

Three payload kinds ship:

- **`tag-like`** — drop onto a Trello card to create a label, onto a Calendar event to prefix its description, onto a Keep note to attach a label.
- **`clip-like`** — click a producer row to bind it to the scratchpad; edit the textarea; on blur the new content writes back to the source.
- **`note-like`** — drag scratchpad text onto a Trello card or Calendar event to append it to the description.

Connectors opt in by registering `payloadAdapters` on their manifest. The retry queue (`POST /api/local/pending-writes`) handles backoff `1s → 5s → 30s → 5m`; permanent 401s surface inline reconnect pills on affected cards.

## Registry codegen

Three indexes are auto-generated by `pnpm gen:registries` from disk state:

- `app/connectors/index.ts`
- `app/tiles/index.ts`
- `app/themes/index.ts` + `app/themes/index.css`

API route shims under `app/api/<connector>/...` are also auto-generated — actual route handlers live under `app/connectors/<id>/api/`.

CI gates:

- `pnpm gen:registries:check` — fails when registries are out of sync with disk.
- `pnpm lint:check` — runs the above plus an orphan-registry audit (workspace quick-pick icons against lucide; payload-kind parity between `types.ts` ↔ `schema.ts`).

Folders prefixed with `_` (e.g. `_template/`, `_shared/`) are excluded from registries.

## Mobile-first constraints

LENS will eventually target React Native (Expo). This shapes every architectural call today:

- **No DB code in components.** Components import types + React Query hooks. Period.
- **Server-only DB layer.** `lib/supabase/*` and `connectors/*/client.ts` never import into client modules.
- **Hooks abstract the data source.** `lib/hooks/use-*.ts` and `connectors/*/hooks/*` hide whether data came from `/api/...` (web) or a native call (mobile).
- **Zustand is UI state only.** Drag preview, panel open state, theme — never data of record.

When we lift to Expo, only the API/fetch layer is replaced. The component layer survives.
